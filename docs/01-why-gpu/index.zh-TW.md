# 第一部分 —— 為什麼需要 GPU

在掀開引擎蓋之前,我們得先回答一個問題:**為什麼 GPU 的形狀,剛好適合 LLM?** 重點不是「它怎麼變快的」,而是「它那種快,為什麼剛好是我們需要的」。

這一部分會先建立三個後面每一章都會用到的觀念:

1. **CPU 與 GPU 的設計目標剛好相反。** 低延遲 vs. 高吞吐量。少數幾個優秀的工人 vs. 一大群同步合作的工人。
2. **LLM 的數學運算是「極度平行」的。** 數十億次彼此獨立的乘法 —— 正好是 GPU 被設計來吞噬的那種形狀。
3. **協調勝過聰明。** GPU 贏過 CPU,不是因為每一個核心都更聰明,而是因為它讓數百萬個核心同步做同一件事。

我們會透過兩個場景來談這些觀念:**餐廳廚房**(用來談平行與延遲),以及**工廠生產線**(用來談吞吐量與管線化)。兩個類比都不完美,所以當某個比喻能讓重點更清楚時,我們就會切換過去。

## 這部分的章節

- [自助餐廳 vs. 牛排館](cpu-vs-gpu.md) —— 為什麼 CPU 像主廚、GPU 像自助餐廳。
- [吞吐量優於延遲](throughput-vs-latency.md) —— 為什麼「一份餐要多久能出?」是拿來問 GPU 時錯誤的問題。
