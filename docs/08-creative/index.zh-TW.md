# 第八部分 —— 創意工作流

這份指南的其他部分是寫給打造 LLM 系統的人。第八部分是寫給使用它們的人 —— 作者、設計師、行銷、製作人、編輯。你不需要大多管線。但你*需要*足夠的心智模型,才能跟工具協作、而不是被它拉著走。

<div class="hero" markdown>

用 AI 做創意工作的陷阱,跟做工程的陷阱不一樣。工程師擔心 hallucinate 的 citation 和 prompt injection。創意人擔心的是:草稿聽起來很 generic、失去聲音、以及太常接受「還 OK」的輸出帶來的慢性品味流失。這一部分在講「用工具而不被工具用」這門手藝。

</div>

## 這部分有什麼

- [給創意人的 diffusion basics](diffusion-basics.md) —— image model 怎麼運作,夠你有意識地 prompt 它。
- [AI 輔助工作裡的風格與聲音](style-and-voice.md) —— 工具在把你拉向平均值時,怎麼守住你的簽名。
- [創意工作流整合](creative-integration.md) —— 真實專案裡好用的實戰 pattern。

## 誰寫的、為什麼

這部分比其他短,也更有立場。這領域在演;具體工具會變;技能不會。我們希望你從這裡帶走的是一種*工作方式*,而不是一張功能列表。

## 一句話帶走

AI 工具壓縮了創意工作裡機械的部分,讓你更多時間可以花在真正創意的部分 —— 但前提是你要*主動守住*那些真正創意的部分,因為工具很樂意用平均值幫你填滿。
