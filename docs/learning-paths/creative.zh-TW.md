# 路徑四 —— 創意工作者

## 這是寫給誰的

你是作者、設計師、行銷、插畫家、製作人或編輯。你為人類做東西。過去兩年,一堆 AI 工具被擺在你桌上 —— Figma 裡的影像生成器、CMS 裡的寫作助理、剪輯器裡的語音工具。你已經過了「要不要試?」的階段,進到「我要怎麼用這些東西用得好,又不會弄丟我的聲音或我的 taste?」。

## 目標

走完這條路徑之後,你會:

- 對這些 model 的運作機制有足夠的理解,可以精準地 prompt,而不是靠許願。
- 有一套能放大你的 taste 的 workflow,而不是取代它。
- 分得出「能過關的 AI 輸出」和「真的好的 AI 輸出」—— 尤其是在自己的作品裡。
- 有自信地面對倫理與職業上的問題(揭露、訓練資料、客戶期待)。

## 先備知識

一項手藝。好奇心。不需要寫程式 —— 但如果你曾經花時間寫很細的 prompt,你會領先一點。

## 這條路徑

### 第一步 —— 一點點(真的一點點)機制

你不需要完整的工程基底。但你需要知道*自己正在跟哪一種機器打交道*,因為你習慣的類比(Photoshop filter、搜尋引擎、自動完成)各自在一些小地方全是錯的。

1. [自助餐廳 vs. 牛排館](../01-why-gpu/cpu-vs-gpu.md) —— 一篇 10 分鐘的讀物,用來校正直覺。
2. [Token 是什麼?](../03-llm-inference/token.md) —— 重要。model 看到的是 token,不是字。
3. [一個 token 的一生](../03-llm-inference/life-of-a-token.md) —— 為什麼輸出越長越會飄,以及為什麼有時候把對話重開會有幫助。

**帶走的重點:** LLM 是「在所有先前 token 條件下,下一個 token 的機率模型」。每一個「聽起來好有把握!」的瞬間,本質就是:一個高機率的 completion,不是一個判斷。

!!! warning "誤解檢查點"

    **「model 寫得很有把握,那它應該知道自己在說什麼。」** 聽起來更接近真的的親戚說法:*「有自信的人通常懂自己的主題。」* 對人類而言,自信和專業有相關。對 LLM 而言,自信和正確*幾乎正交* —— 不管內容是真是假,model 都會產出語法上有把握的文字。「流暢」是預設狀態,「準確」是另一件你得去驗證的事情。

### 第二步 —— 把 prompting 當成創作實踐

新章節(和開發者路徑共用):[Prompt engineering 模式](../06-building/prompt-engineering.md)。JSON-schema 那部分可以跳過;把時間花在:role conditioning、reference 範例、迭代與修改的 prompt、persona 保持、風格轉換,還有「show, don't tell」式的 prompting。

幾個創意工作者特別會想掌握的模式:

- **Reference 先行:** 在描述任務*之前*,先貼上 2–3 個你想要的聲音/調性範例。
- **約束式 prompt:** 緊的規則(「不要用分號;每句話最多 12 字;一段只能有一個 metaphor」)會比鬆的指示產出更受控的結果。
- **修改,不要重生:** 跟 model 說「把第三段改得更具體」,不要要求它整篇重寫。你才留得住有在發揮的部分。
- **兩趟式:** 用一個 model 寫草稿,用另一個(或同一個搭配不同的 prompt)做批評。那一趟批評常常才是抓出爛寫作的關鍵。

**帶走的重點:** Prompting 是一門手藝,不是什麼 hack。擅長的人,就是擅長描述他們到底想要什麼。

!!! warning "誤解檢查點"

    **「好 prompt 就是找到那些魔法字。」** 聽起來更接近真的的親戚說法:*「搜尋引擎會獎勵對的關鍵字。」* 在 SEO,沒錯。但在 LLM 上,「魔法字」式 prompting(「你是一位具有 30 年經驗的小說家,請以你職涯存亡全賴此文的態度回答」)比起*把你要的工作講清楚*,帶來的邊際收益小到看不見。真正的槓桿在:給具體範例、設具體約束、迭代。「prompting 大師」這個角色有一半是都市傳說。

### 第三步 —— 生成影像:另外一種機器

新章節:[給創意工作者的 diffusion 基礎](../08-creative/diffusion-basics.md)。Diffusion model 實際上在做什麼(從 noise 到影像)、為什麼 Midjourney/SD 的 prompt 語法跟 LLM prompt 不一樣、以及你會用到的詞彙:prompt weighting、reference 影像、ControlNet、seed、長寬比、風格 model、inpainting/outpainting。

這一章不是完整的工程解釋 —— 它是一份*使用者心智模型*,好讓你的 prompt 少一點抽獎感。

**帶走的重點:** Diffusion model 是靠*從隨機雜訊一步步去雜*來生成的。這也是為什麼 seed 重要(不同起始雜訊 → 不同影像)、為什麼小小改 prompt 可以讓整張圖大洗牌、以及為什麼 inpainting 效果那麼好(其他部分凍住,只對 mask 重新去雜)。

!!! warning "誤解檢查點"

    **「影像 model 是在訓練集裡『搜尋』一張符合的圖。」** 聽起來更接近真的的親戚說法:*「自動完成是從 corpus 裡 retrieve 結果。」* Google 圖片搜尋做的是那件事。Diffusion model 不做 retrieval —— 它是*生成*:在 prompt embedding 的引導下,對一個隨機 tensor 反覆去雜。這也是為什麼它會產出不在訓練集裡的新影像、為什麼同一個 prompt 配不同 seed 會得到不同結果、以及為什麼會出現那些「幾乎像人但又不對」的詭異時刻 —— 那是生成失敗,不是 retrieval 失敗。

### 第四步 —— 保住你的聲音

新章節:[AI 輔助創作中的風格與聲音](../08-creative/style-and-voice.md)。保留(以及稽核)聲音的技術:把 style guide 當 prompt、校準樣本、先改寫再 diff、什麼時候該直接扔掉一份「還行但很普通」的草稿。

有一節專門講那個診斷性問題:*如果在一個作者混列裡,我認得出這是我寫的嗎?* 以及誠實回答這題的工具與習慣。

**帶走的重點:** AI 輸出會往平均值收斂。你的工作是跟這種收斂對抗 —— 一邊用 prompt 把它推開,一邊在編輯時把它拉回自己。這個工具放大的是使用者。如果「使用者」很普通,輸出就會很普通。

!!! warning "誤解檢查點"

    **「AI 寫的東西一聽就是 AI。」** 聽起來更接近真的的親戚說法:*「機器翻譯以前聽起來就明顯不對。」* 那在 2018 年是對的。現在不再是。一份稍加編輯的 LLM 草稿,對隨意讀者來說通常跟人類寫作分不出來 —— 但它會傾向於一種特定*味道*(和氣、保守、結構工整、愛用三項並列)。如果你用 default,你的寫作會慢慢漂向那個味道。好的 AI 使用是主動*抗拒* default,這需要夠強的個人風格,加上編輯的耐心。

### 第五步 —— Workflow 與整合

新章節:[創意 workflow 整合](../08-creative/creative-integration.md)。常見任務的具體 workflow:研究 → 大綱 → 草稿 → 編輯;用影像 model 做 storyboard 和 mood board;劇本 → 視覺 → 配音的 pipeline;帶品牌約束的行銷文案變體;搭配 AI 的短影音。

能工具無關的地方就盡量無關,並附上目前各類別領先者的註記。

**帶走的重點:** 贏面不是「AI 幫我寫整篇文章」。贏面是「AI 把我不愛做的部分壓縮掉,我就能把時間花在我愛的部分上」。草稿變體和機械性的初稿壓得很好。最後的潤飾、聲音、判斷壓不了。

### 第六步 —— 倫理、歸屬、客戶期待

這塊的答案整個領域都還沒定下來。但有幾條還算穩定的大原則:

- **該揭露的時候揭露。** 如果客戶付錢買的是你的*聲音*(ghostwriting、個人散文),用 AI 通常是要主動說的事。如果你只是用 AI 加速機械性工作(初稿翻譯、背景研究),大多數客戶不會預期你要揭露。
- **搞清楚你的訓練資料情況。** 那些「從 opt-out 內容訓練」的影像 model 一直在變。商用許可依供應商不同。條款要讀。
- **對自己誠實地面對作者身份。** 如果 model 寫了一篇 90%,你的 byline 應該反映這件事 —— 先對自己。創意工作的長青來自於真的擁有自己的作品。

!!! warning "誤解檢查點"

    **「用 AI 本身就是作弊。」** 聽起來更接近真的的親戚說法:*「捷徑會貶低手藝。」* 每一個創意媒介都吸收過新工具 —— 攝影取代了肖像畫、數位取代了暗房、Photoshop 取代了噴槍、auto-tune 取代了重錄。當機械性的那一層被壓縮,手藝會整體*往上移*。真正的問題不是「你是不是用了工具?」,而是「你帶進來的、工具帶不進來的是什麼?」如果答案是「沒有」,那件作品本來就很可能很普通。如果答案是「taste、聲音、選擇、修改、脈絡」,那工具就只是工具。

### 第七步 —— 想再深一點(可選)

如果你想了解引擎蓋底下在發生什麼:

- [慢慢講清楚 Attention](../03-llm-inference/attention.md) —— model 在生成下一個 token 時如何「關注」先前的 token。短、好讀。
- [KV 快取](../03-llm-inference/kv-cache.md) —— 為什麼很長的對話有時會變慢或變怪。技術性較高,可以掃過去。

## 誤解總表

| 誤解 | 它從哪裡來 |
|---|---|
| 有自信 = 對 | 人類社交線索不適用在 model 上。 |
| 魔法 prompt 字 | SEO 式的關鍵字思維。 |
| 影像 model 做 retrieval | 把生成和搜尋混在一起。 |
| AI 寫的東西一聽就是 AI | 2020 年代工具留下的過時經驗。 |
| 用 AI 就是作弊 | 新工具焦慮;歷史上很常見。 |

## 檢查點

在你說這條路徑走完之前:

1. 有人請你用你的招牌聲音寫一篇 1,000 字的業配文。走過一次你會怎麼使用 AI 協助,而又不弄丟你的聲音。
2. 對一位非技術背景的同事,描述為什麼 Midjourney 有時會在不同 prompt 下給你幾乎一樣的構圖,而有時又在同一個 prompt 下給你天差地遠的結果。
3. 一個客戶問:「這個你有用 AI 嗎?」你會怎麼回答,以及回過頭看,你會希望怎麼回答?
4. 你寫了一篇自己滿意的稿子,編輯卻說它聽起來有 AI 味。你用什麼 checklist 來診斷?
5. 列出三件你*不會*用 AI 做的事情,以及為什麼。

如果你答得出來,你就是在用工具寫作,而不是被工具寫出來。
