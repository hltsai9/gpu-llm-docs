# 第七部分 —— 策略與產品

第一到第六部分在講 LLM *在做什麼*。第七部分在講它們是*為了什麼*、誰來做那些決定、以及怎麼做這些決定才不會過度承諾或用得太少。

<div class="hero" markdown>

挑對要做的功能、定對價、把範圍對上 model 真實的能力、為失敗模式做規劃 —— 這些技能比任何特定 model 都活得長。大多糟糕的 AI 產品失敗,不是因為技術差,而是因為圍繞在它外面的產品思維太模糊 —— 一個讓主管驚艷的 demo 變成一份忽略成本、安全、以及「通常會動」和「可靠地會動」之間差異的規格。

</div>

## 這部分有什麼

- [Capability mapping](capability-mapping.md) —— 前沿 model 可靠能做、不可靠能做、和做不到的事。
- [LLM 產品的經濟學](economics.md) —— 按 token 計價、自建 vs. 買 vs. fine-tune、單位經濟。
- [風險與治理](risk-governance.md) —— hallucination、prompt injection、隱私、監管。

## 為什麼這部分存在

產品負責人常被「不知道自己在問什麼的人」要求出 AI 功能,還要向「不想聽到 AI 限制的人」解釋那些限制。這裡的詞彙和結構是為那個中間位置設計的 —— 足以讓工程師信服的機制,足以讓主管信服的商業框架,還有足以避免可避免失敗的誠實。

## 一句話帶走

一個好的 LLM 產品決策有三條腿:*技術能不能可靠做到這件事*、*以我們的流量做這件事付不付得起*、*我們能不能接受它的失敗模式*。任何一條腿弱了,這功能就是一個 liability,不管 demo 多漂亮。
