# 第三部分 —— LLM 推論

廚房已經就定位。現在我們站在旁邊,看它怎麼煮。

第三部分會跟著一個 token 走完整段旅程:從你按下「送出」,到它出現在螢幕上。一路上我們會遇到 Transformer 的核心運算、讓串流輸出成為可能的 KV 快取,以及推論的兩個截然不同的階段 —— prefill 和 decode,各自有各自的效能個性。

讀完之後,你會明白為什麼「吞吐量」和「延遲」是兩個不同的指標、為什麼伺服器要這麼積極地批次處理請求、為什麼 `max_seq_len` 和 `batch_size` 是大家爭個不停的兩個旋鈕,以及「decode 階段是受限於記憶體」這句話到底意味著什麼。

## 觀看角度的切換

第二部分是「硬體先行」:*這是一顆 GPU,它是這樣運作的。* 第三部分是「工作負載先行」:*這是一個 Transformer 如何跑,以及它怎麼跟硬體撞在一起。* 兩邊會在一個叫做 **Roofline(屋頂線)** 的觀念裡交會 —— 運算和記憶體,誰先用光誰就是效能的天花板。

<div class="analogy" markdown>
想像你站在旁邊,看著一張單子從點單、進廚房、到擺盤端出去的全程。第三部分就是這趟導覽。沿途每一站都在做特定的事、有各自的節奏,也有各自的瓶頸。
